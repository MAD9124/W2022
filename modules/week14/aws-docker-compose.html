# AWS ECS with Docker Compose

# Set AWS Access Keys

Set up the local config and credentials files for AWS CLI access.

mkdir ~/.aws
touch ~/.aws/config
touch ~/.aws/credentials

Copy this text into the config file.

[default]
output = json
region = us-east-1

Get you AWS Classroom account credentials.

Click the account details button and then on the next screen, click the show button.

Copy the displayed content and paste it into the ~/.aws/credentials file. It will look something like this ...

[default]
aws_access_key_id=<YOUR_ACCESS_KEY>
aws_secret_access_key=<YOUR_SECRET_KEY>
aws_session_token=<A_VERY_LONG_STRING>

# Create a Docker Context

Docker uses context profiles to know where to launch new container instances. The default context points to your local machine. Using the docker context create ecs command, you can create a new one that will point to your AWS Classroom account. The command will prompt you with two questions. Choose An existing AWS profile and default respectively.

docker context create ecs mad9124
? Create a Docker context using: An existing AWS profile
? Select AWS Profile default
Successfully created ecs context "mad9124"

Verify your settings by running the docker context ls command.

Switch the active Docker context to your new "mad9124" profile.

docker context use mad9124

Create a docker-compose.yml file.

version: '3.8'
services:
  express:
    image: <YOUR_DOCKER_HUB_USER_NAME>/giftr-api-w21
    container_name: giftr-api-w21
    environment:
      API_JWTKEY: esftq9gj691gw18dmn2myul1nz9hqy
      API_PORT: 80
      NODE_ENV: 'stage'
    ports:
      - 80:80
    command: node server.js
    deploy:
      x-aws-autoscaling:
        min: 1
        max: 3
        cpu: 70

Deploy your application to AWS.

docker compose up -d

It will take a few minutes to create the virtual network and spin-up your containers. Check on the deployment progress:

docker compose ps

When it is running, you will see the URL for the load-balancer displayed in the output.
